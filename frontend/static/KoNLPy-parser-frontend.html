KoNLPy parser front-end Notes.

 

Here are details for the KoNLPy-based Korean parser app I showed several people at the Korean Language Meetup  on Saturday.  I have it running on an AWS instance so you can try it out. the URL is:

 

            http://ec2-52-34-61-205.us-west-2.compute.amazonaws.com/analyzer 

 

Key in a sentence (or several) and hit the 'parse' button, it should draw a parse tree after a second or so.  The codes in parentheses after each word are part-of-speech tags generated by the back-end parser, explained below.  Click on a word to bring up an embedded iframe containing a search for that word on Wiktionary (which has entries for particles and endings and other parts of speech, as well as whole words.)

 

In its current form, the displayed tree is a *very* crude approximation to phrase structures in Korean; that's one of the things I want to improve, and perhaps Yoo Jin or Philjae can help with that.  More on that below.   

 

Note that it's just running a single-threaded dev server, so certainly not prod ready, but it should be fine for trying out.

 

Sources

 

The source is up on github here - https://github.com/johnw3d/Korean-language-parser.  It should be public; let me know if you can't get access.   The front-end is written in Vue js organized with webpack and the back-end is written in Python using the Flask server framework.  I need to update the README to explain dev setup and build instructions, will do so soon.  

 

Parser back-end

 

The app is pretty simple really, just a UI that wraps the KoNLPy framework describe here - http://konlpy.org/en/latest/.  Vue is pretty cool for rendering SVG, so it wasn't hard to knock up the graphics.  The KoNLPy framework itself wraps a set of 4 or 5 3rd-party parsers, listed in the docs.  The app I have up on AWS is using the Kkma parser, but I plan on making that selectable.  I've only just started playing with them and not sure which is the best.

 

The parsers themselves only separate the sentence into a list of words tagged with their parts-of-speech, using codes listed in this spreadsheet - https://docs.google.com/spreadsheets/d/1OGAjUvalBuX-oZvZ_-9tEfYD2gQe7hTGsgUpiiBSXI8/edit#gid=0.  These are the codes in parentheses after each word in the graph.  The phrase tree is generated using the chunking parser in NLTK, a popular NLP Python library.  You give it a set of regular-expressions defining phrases in terms of pos-tag patterns, as described here: http://www.nltk.org/book/ch07.html - section 2 - chunking grammars.  What we'd need is to get help from Yoo Jin & Philjae to help decode the parts-of-speech tags in the spreadsheet linked above and say what sequences of these tags form the various phrase constructs in Korean.
